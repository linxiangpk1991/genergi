# 2026-04-20 UI 动效与排版热修复说明

Updated: 2026-04-20

## 1. 版本标识

- 仓库：`linxiangpk1991/genergi`
- 分支：`main`
- 发布方式：`pnpm deploy:hotfix -- <explicit file list>`
- 线上入口：[https://ai.genergius.com](https://ai.genergius.com)

这次不是一次新的 UI 大改版，而是在保留 Gemini 已落地视觉方案的前提下，对两个已经确认的体验问题做最小热修：

- 修正正文/辅助文案被误套用为 eyebrow 小标签排版
- 修正运行中状态点的持续动效没有 reduced-motion 兜底，且使用了高成本 `box-shadow` 动画

---

## 2. 热修范围

### 2.1 排版范围收口

本次把 `apps/web/src/styles.css` 中过宽的选择器拆开处理：

- 保留 `.eyebrow` 的 12px / uppercase / Latin label 风格
- 让 `.topbar p`
- `.muted`
- `.task-item span`

回到普通辅助文案语义，不再共享 eyebrow 的标签型排版

这次修复的目标不是推翻现有视觉方向，而是避免中文运营侧说明文案被渲染成过小、过紧、可读性下降的“伪标签文字”。

### 2.2 运行中状态点动效收口

本次保留了最近活动区域“任务运行中”的视觉提示，但把实现改成了更轻量的版本：

- `pulse` 从 `transform + box-shadow` 改为 `transform + opacity`
- 增加 `@media (prefers-reduced-motion: reduce)` 分支
- 在 reduced-motion 下关闭 `.status-dot--running` 动画

这样做的结果是：

- 视觉上仍保留轻微的“正在运行”提示
- 减少持续重绘带来的不必要性能负担
- 尊重系统级降低动态效果偏好

## 3. 文件级改动

### 3.1 实际代码文件

- `apps/web/src/styles.css`
  - 拆分 eyebrow 与正文/辅助文案选择器
  - 调整 `pulse` 动画实现
  - 增加 reduced-motion 兜底

### 3.2 回归测试

- `tests/unit/web/ui-style-regressions.test.ts`
  - 防止 `.eyebrow` 再次误伤 `.topbar p` / `.muted` / `.task-item span`
  - 防止 `pulse` 再次回到 `box-shadow` 动画
  - 防止 reduced-motion 兜底再次丢失

## 5. 对团队的意义

这次热修的意义不是“多做了一层 polish”，而是把已经落地的 UI 方案从“看起来更精致”进一步收口成“更能长期维护”的状态：

1. 不再让排版优化误伤大量中文辅助文案
2. 不再让运行中提示依赖高成本动效
3. 对 reduced-motion 用户给出明确尊重
4. 用回归测试把这两个问题钉住，避免后续视觉微调时重复引入

---

## 6. 后续建议

建议后续继续补三件事：

1. 给其余首页交互动效补统一的 reduced-motion 审视
2. 对首页和登录页做一次浏览器级视觉回归截图基线
3. 把这类“UI 热修 + 回归测试 + 显式文件 hotfix”沉淀成固定发布模板
