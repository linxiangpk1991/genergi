# GENERGI 仓库交接说明

Updated: 2026-04-17

## 1. 项目是什么

`GENERGI 自动化视频平台` 是一个面向海外英文短视频生产的 web-first 平台。

当前目标：

- 面向 TikTok / Instagram Reels / YouTube Shorts
- 中文后台操作，英文内容输出
- 支持从任务发起到成片导出的完整链路

## 2. 当前仓库边界

真实项目根目录与公开仓库母体：

- `E:\genergi`

参考旧仓库：

- `E:\short-video-factory`

说明：

- 旧仓库仅作历史参考，不再作为新的开发母体。
- 如果未来某个线程 UI 仍显示旧目录，以当前仓库和 `AGENTS.md` 为准。

## 3. 当前架构

- `apps/web`：React + Vite 控制台
- `apps/api`：Hono API
- `apps/worker`：BullMQ 执行引擎
- `packages/shared`：任务 / 资产 / 用户持久化与共享 schema
- `packages/config`：品牌、模式、渠道、模型配置
- `packages/ui`：设计系统基础

当前阶段仍以文件持久化为主，默认写入：

- `.data/`

生产或共享环境建议显式设置：

- `GENERGI_DATA_DIR`

## 5. 本地接手建议

### 环境

- Node.js `>= 24`
- pnpm `>= 10.12.4`
- Redis 可选

### 启动顺序

```bash
pnpm install
pnpm dev
```

按应用单独启动：

```bash
pnpm dev:web
pnpm dev:api
pnpm dev:worker
```

### 本地运行注意点

- 未配置 `REDIS_URL` 时，worker 会停止队列消费，这是当前允许的本地 bootstrap 行为。
- 未显式设置 `GENERGI_DATA_DIR` 时，任务与用户数据默认写入仓库下 `.data`。
- 开发环境允许存在本地登录兜底逻辑，但任何共享、测试、预发、生产环境都必须使用显式环境变量配置。

## 8. 部署模型

正式环境目录模型：

```text
/opt/genergi/
  releases/<timestamp>/
  current
  current.prev
  shared/
```

关键规则：

- release 目录只放代码与构建产物
- 运行期数据必须放到稳定共享目录
- `apps/web/dist` 由 `nginx` 提供
- `api` / `worker` 由 `systemd` 守护

参考文档：

- `docs/architecture/生产部署模型.md`
- `docs/architecture/稳定部署能力说明.md`
- `docs/architecture/deployment-baseline.md`

## 9. 当前优先级

默认建议优先级：

1. 稳定图片直出链
2. 增强用户中心与登录体验
3. 持续提升部署可靠性
4. 把资产中心继续做成更完整的预览 / 导出工作台

## 10. 仓库安全规则

- 不提交真实 API key、密码、token、session secret
- 不提交私有 provider endpoint
- 不提交服务器上的私有环境文件
- 不提交本地工具目录，例如 `.claude/`
- 任何生产接入信息都通过环境变量或私有部署配置注入
