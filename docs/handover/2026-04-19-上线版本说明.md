## 2. 本次上线范围

### 2.1 Workbench UI/UX 收口

已上线的控制台界面调整包括：

- 侧边栏导航改成内容生产语义：
  - `任务启动`
  - `分镜审阅`
  - `关键帧审阅`
  - `生产看板`
  - `交付资产`
  - `用户中心`
- 首页默认入口改为“内容母本优先”，移除不必要的技术黑话和假入口
- 批量看板只展示真实任务、真实运行状态和需要人工处理的异常
- 资产中心把目标时长、实际时长、偏差、容差判断和交付就绪度提到顶层
- 用户中心补齐搜索、筛选、ID 复制与更清晰的操作层级

### 2.2 审阅写回闭环

这是本次版本最重要的能力变更。

已上线：

- `Storyboard Review` 支持真实提交：
  - `通过`
  - `退回`
- `Keyframe Review` 支持真实提交：
  - `通过`
  - `退回`
- 审阅备注会一起写回服务端
- 服务端返回更新后的：
  - `TaskSummary`
  - `TaskDetail`
- 前端收到返回结果后会立刻刷新当前页面
- 审阅通过后，页面会自动跳到下一个待审 scene

### 2.3 审阅状态传播

这次不只是把按钮点通，而是把状态传播到了下游页面。

现在这些页面都会看到真实 review 状态：

- `分镜审阅页`
- `关键帧审阅页`
- `批量任务看板`
- `资产中心`

可见信息包括：

- 当前 review 阶段
- 待审数量
- 任务是否进入 `waiting_review`
- 资产页是否仍处于 `待审分镜 / 待审关键帧 / 自动 QA`

## 3. 代码层改动摘要

### 3.1 Shared / Persistence

本次新增或扩展了 review 相关共享契约：

- `reviewDecisionInputSchema`
- `reviewDecisionBodySchema`
- `reviewSummarySchema`

并扩展了：

- `TaskSummary`
  - `reviewStage`
  - `pendingReviewCount`
  - `reviewUpdatedAt`
- `TaskDetail`
  - `reviewStage`
  - `pendingReviewCount`
  - `reviewUpdatedAt`
- `StoryboardScene`
  - `reviewNote`
  - `reviewedAt`
  - `keyframeReviewNote`
  - `keyframeReviewedAt`

同时补了：

- 旧任务记录的 review 字段默认值迁移
- seed summary 的 review 初始值
- detail 归一化逻辑

### 3.2 API

新增真实写回接口：

- `POST /api/tasks/:taskId/reviews/storyboard_review/:sceneId`
- `POST /api/tasks/:taskId/reviews/keyframe_review/:sceneId`

请求体最小化为：

- `decision`
- `note?`

接口返回：

- 更新后的 `task`
- 更新后的 `detail`

### 3.3 Worker / Planner 保留链

为了避免 review 元数据在 scene 重建时被覆盖，本次还补了：

- scene review 元数据 merge 逻辑
- `scene.id` 优先、`index` 次级兜底的保留策略
- `rewriteTaskWithTextProvider()` 中 review 字段保留
- task detail round-trip 时 review 元数据保留

这一步很关键，否则前端虽然能写回，但 worker 或 task-store 一归一化就会把 review note / timestamp 冲掉。

## 5. 当前仍然存在的非阻塞项

这次版本已可用，但还有一些不是 blocker 的后续项：

### 5.1 关键帧缺失时会出现真实空态

对没有真实关键帧产物的历史任务：

- 页面会展示正确的空态说明
- 浏览器控制台会出现对应的 keyframe 404

这不影响当前版本功能正确性，但后续可以再补一层“预览前先判断 keyframe 资产存在性”的优化，减少控制台噪音。

### 5.2 高级审阅动作仍未上线

以下动作仍然保持禁用：

- 合并 Scene
- 拆分 Scene
- 升级模型重试
- 返回脚本调整
- 重新生成建议 / 图片

这是刻意保持诚实，不是遗漏。  
本次版本只上线已经真正接通的审阅写回，不伪装未完成能力。
