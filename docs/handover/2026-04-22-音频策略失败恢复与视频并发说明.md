# 2026-04-22 音频策略、失败恢复与视频并发说明

## 本次目标

这次版本聚焦三个方向：

1. 把任务创建扩展为可选择音频策略。
2. 给失败任务增加真实可用的恢复运行入口。
3. 把多分镜视频生成从串行改成受控并发，缩短 30s / 45s 任务等待时间。

## 已完成内容

### 1. 音频策略正式接入任务链路

- 新增两档音频策略：
  - `tts_only`
  - `native_plus_tts_ducked`
- 任务创建页新增可选按钮：
  - `TTS 主导`
  - `原生音频 + TTS 混音`
- 音频策略会冻结到任务配置，不会在任务运行后漂移。
- 审核页、素材资产中心、生产看板都已补充音频策略展示，便于排查当前任务到底走的是哪条音频链。

### 2. Veo 原生音频保留并混音

- 已确认 `Veo 3.1` 生成返回的 `scene mp4` 自带原生音轨。
- 原先链路会在 `trimmed-scenes.mp4` 阶段通过 `-an` 删除原音，再强制换成 TTS。
- 现在当任务音频策略为 `native_plus_tts_ducked` 时：
  - 裁切阶段保留视频原音
  - 最终输出使用 `amix` 将原生音轨压低后与 TTS 旁白混合
- 若混音失败，会自动回退到 narration-only 的稳定链路，不阻塞任务完成。

### 3. 失败任务恢复运行

- 新增真实恢复接口：
  - `POST /api/tasks/:taskId/resume`
- 恢复不是前端假动作，而是：
  - 清空失败原因
  - 清空取消标记
  - 把任务状态回置到 `queued`
  - 文案改为 `等待 worker 恢复处理`
  - 重新入队
- 对已经过审核蓝图后失败的任务，会自动使用：
  - `continueExecution = true`
  避免错误地从头重建蓝图。
- 前端已补按钮位置：
  - 生产看板失败任务卡片
  - 素材资产中心右侧当前处理入口

### 4. 多分镜视频生成改为受控并发

- 原先 `createSceneVideoBundle()` 使用 `for...of + await`，属于严格串行。
- 现在改为受控并发执行，默认并发数：
  - `4`
- 新增环境变量：
  - `GENERGI_VIDEO_SCENE_CONCURRENCY`
- 保护策略：
  - 最小值 `1`
  - 默认值 `4`
  - 最大值 `6`
- 返回结果仍按 `scene.index` 保序，保证后续 concat / trim / mux 顺序稳定。
