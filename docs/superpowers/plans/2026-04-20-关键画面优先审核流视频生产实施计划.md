# 关键画面优先审核流视频生产实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 把 GENERGI 从“文字直接出视频、关键画面事后补产物”的链路，升级为“执行蓝图先生成、关键画面先产出、审核流整任务放行、通过后再生成完整分镜视频”的生产平台。

**Architecture:** 先新增项目、终端尺寸预设、执行蓝图、蓝图版本与审核记录的共享契约和持久化，再把任务创建与 worker 执行链切成“全自动 / 审核流”两条主线。审核流先卡在“蓝图 + 关键画面”阶段，审核通过后再通过队列恢复下半段执行；图片模型前移为视频生成的直接输入，项目级审核库只沉淀审核通过版本。

**Tech Stack:** TypeScript, React, Hono, shared Zod contracts, file-based persistence, BullMQ worker, FFmpeg, Vitest
